# 迭代需求：关卡编辑、任务系统与本地排行榜

## 背景

当前 `霓虹跃迁` 只是一个最小可玩的点击反应小游戏。下一轮需要把它升级为具备“玩法循环 + 关卡配置 + 任务成长 + 本地存档”的完整游戏样板。

## 核心目标

让项目从单场景点击游戏进化为可配置、可成长、可重复游玩的轻量游戏框架，覆盖 UI、场景管理、游戏状态和本地存档。

## 需求范围

1. 增加开始界面、游戏主场景、结算界面、排行榜界面，形成完整场景流转。
2. 引入关卡配置数据，至少支持：
   - 不同波次刷新速度
   - 不同目标类型
   - 不同得分规则
   - 关卡胜负条件
3. 增加任务系统，例如：
   - 单局达到指定分数
   - 连续命中若干目标
   - 在限定时间内完成挑战
4. 增加玩家生命值、连击数、道具或技能中的至少两项机制。
5. 增加本地排行榜或本地存档能力，刷新页面后仍能看到历史最高分、完成任务记录和解锁关卡。
6. 将游戏逻辑、UI 层、场景切换、配置数据拆分成多个模块，不能继续把所有逻辑塞在一个文件里。
7. 如果实现顺手，可增加键盘控制或移动端兼容逻辑。
8. 如引入资源文件，需保持项目结构清晰。

## 技术要求

- 保持 Phaser + TypeScript 技术栈
- 需要真实的场景切换和状态管理
- 存档至少使用 `localStorage` 或其它本地持久化方案
- 代码结构要适合继续扩展更多玩法

## 验收重点

- 游戏不再是单场景演示
- 玩法机制、UI、状态和存档都有明显扩展
- 文件拆分合理，具备继续增加关卡和道具的空间
- 本地运行体验稳定
